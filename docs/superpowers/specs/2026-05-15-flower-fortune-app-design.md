# 花占いアプリ 設計仕様書

作成日：2026-05-15

---

## 概要

生年月日と血液型をもとに、毎日異なる花を提示してその花言葉にちなんだ占いを行うiOSアプリ。詳細な占い結果は課金（月額サブスクまたは都度課金）で解放される。サーバーなし・モバイル完結。

---

## ターゲット

- プラットフォーム：iOS（App Store販売）
- 開発フレームワーク：React Native / Expo SDK 51+

---

## アーキテクチャ

```
Expo (React Native) / iOS
├── オンボーディング画面
│   └── 生年月日 + 血液型を入力 → AsyncStorageに保存
├── ホーム画面（毎日更新）
│   ├── 今日の花（画像 + 名前）※無料
│   ├── 花言葉　※無料
│   ├── 一言占い結果　※無料
│   └── 「詳細を見る」ボタン → 課金ゲート
├── 詳細占い画面
│   ├── 恋愛運 / 仕事運 / 金運 / 健康運
│   └── ラッキーカラー・ラッキーアイテム
├── 課金画面
│   ├── 月額プラン（480円/月）
│   └── 都度プラン（120円/回）
└── 設定画面
    ├── プロフィール編集
    └── 購入の復元
```

**データ管理（すべてローカル・サーバーなし）**

| データ | 保存先 |
|---|---|
| 生年月日・血液型 | AsyncStorage |
| 花・占いマスターデータ | アプリ内JSファイル |
| 課金状態（サブスク） | RevenueCat ローカルキャッシュ |
| 都度チケット枚数 | AsyncStorage |

---

## 花・占いの決定ロジック

**シード計算（決定論的・ランダムではない）**

```
シード値 = 生年月日の数値 + 血液型係数 + 今日の日付（YYYYMMDD）
花インデックス = シード値 % 花の総数
```

- 同じユーザーは同じ日に必ず同じ花が出る
- 翌日になると花が変わる

**季節の重み付け**

- 現在の月に対応する季節の花が出やすい（重み：70%）
- 季節外れの花も稀に出る（重み：30%）
- 春：桜・チューリップ・スミレ など
- 夏：向日葵・朝顔・紫陽花 など
- 秋：コスモス・菊・金木犀 など
- 冬：椿・水仙・ポインセチア など

**花データ（約50〜100種類）**

各花に以下を紐付け：
- 名前・画像（ローカルアセット・イラスト調）
- 花言葉
- 無料：一言占いテキスト（1文）
- 有料：恋愛運・仕事運・金運・健康運・ラッキーカラー・ラッキーアイテム

## 課金設計

**プラン構成**

| | 月額プラン | 都度課金 |
|---|---|---|
| 種類 | Auto-Renewable Subscription | Consumable IAP |
| 価格 | 480円/月 | 120円/回 |
| 権利 | 月末まで詳細見放題 | 1回分を消費 |
| 復元 | 可能 | 不可（消耗品） |

**課金フロー**

```
「詳細を見る」タップ
  ↓
月額加入済み？ → そのまま表示
  ↓ NO
都度チケット残あり？ → 1枚消費して表示
  ↓ NO
課金画面を表示
  └ 月額 or 都度を選択 → Apple決済 → 表示
```

**ライブラリ：RevenueCat**

- サブスク状態管理・レシート検証をRevenueCatに委託
- 売上$2500未満は無料プラン適用（初期コスト0）

---

## 画面フロー

```
初回起動
  └→ オンボーディング（生年月日・血液型入力）→ ホーム

2回目以降
  └→ ホーム（直接）
       ├→ 詳細占い画面（課金ゲート通過後）
       ├→ 課金画面
       └→ 設定画面
```

---

## UIデザイン方針

- テーマ：ダーク（神秘的・上品）
- 花画像：イラスト調（世界観統一のため写真不使用）
- フォント：細めのセリフ体
- アニメーション：花がふわっと現れる演出（ホーム初回表示時）
